The `jreactive-ui` artifact is a separate Maven module that provides a set of ready-to-use UI components. Each component is a plain `HtmlComponent` subclass — you use them exactly like any other component: as XML tags in a parent template.

## Adding the dependency

Add `jreactive-ui` to your project's `pom.xml`:

```xml
<dependency>
    <groupId>com.ciro</groupId>
    <artifactId>jreactive-ui</artifactId>
    <version>0.0.1-SNAPSHOT</version>
</dependency>
```

<Note>
  The `jreactive-apt` annotation processor is already pulled in transitively. You do not need to declare it separately for the UI module.
</Note>

## Customization

All UI components respect JReactive's scoped CSS system. You can:

- **Override `template()`** — subclass any component and return your own markup. Every `@Bind` field you declare on the parent class is still available as a template variable.
- **Override CSS classes** — each component uses prefixed class names (`jrx-btn`, `jrx-field`, `jrx-table`, etc.). Target them in your own scoped CSS file to restyle without touching the component source.

```java
// Custom button that adds an icon prefix
public class IconButton extends JButton {

    @Bind public String icon = "";

    @Override
    protected String template() {
        return """
            <button type="button" class="jr-btn {{type}}" @click="{{onClick}}">
              <span class="icon">{{icon}}</span>
              <span>{{label}}</span>
            </button>
            """;
    }
}
```
