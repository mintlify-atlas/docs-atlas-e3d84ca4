---
title: Quickstart
description: Add JReactive to a Spring Boot project and build your first reactive component in minutes.
---

This guide walks you through adding JReactive to an existing Spring Boot project, creating your first component, and running the application.

<Note>
JReactive requires **Java 21** and **Spring Boot 3.2+**. Make sure your project targets Java 21 (`<source>21</source>`) before continuing.
</Note>

<Steps>

<Step title="Add the Maven dependency">

Add `jreactive-starter-spring` as a dependency and configure the `maven-compiler-plugin` to run the APT annotation processor. The processor pre-compiles templates and CSS scoping at build time.

<CodeGroup>

```xml pom.xml — dependency
<dependency>
    <groupId>com.ciro</groupId>
    <artifactId>jreactive-starter-spring</artifactId>
    <version>0.0.1-SNAPSHOT</version>
</dependency>
```

```xml pom.xml — compiler plugin
<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-compiler-plugin</artifactId>
    <configuration>
        <source>21</source>
        <target>21</target>

        <annotationProcessorPaths>
            <path>
                <groupId>com.ciro</groupId>
                <artifactId>jreactive-apt</artifactId>
                <version>0.0.1-SNAPSHOT</version>
            </path>
        </annotationProcessorPaths>

        <fork>true</fork>
        <compilerArgs>
            <arg>--add-exports</arg>
            <arg>jdk.compiler/com.sun.source.util=ALL-UNNAMED</arg>
            <arg>--add-exports</arg>
            <arg>jdk.compiler/com.sun.source.tree=ALL-UNNAMED</arg>
            <arg>-parameters</arg>
        </compilerArgs>
    </configuration>
</plugin>
```

</CodeGroup>

The `jreactive-starter-spring` artifact pulls in `jreactive-core`, `jreactive-runtime-jvm`, and `jreactive-runtime-js`, along with Spring Boot's web and WebSocket starters. Auto-configuration is applied automatically — no `@EnableJReactive` annotation is required.

</Step>

<Step title="Create the Spring Boot application">

Your application class is a standard `@SpringBootApplication`. No additional setup is needed.

```java JReactiveApplication.java
package com.example.myapp;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
public class JReactiveApplication {

    public static void main(String[] args) {
        SpringApplication.run(JReactiveApplication.class, args);
    }
}
```

The `JReactiveAutoConfiguration` bean registered by the starter installs the `AstComponentEngine`, sets up the `PageResolver`, configures the WebSocket hub at `/jrx` via SockJS, and creates a default in-memory `CaffeineStateStore`.

</Step>

<Step title="Create a layout">

A layout wraps every page with shared HTML — the `<head>`, navigation, and the three required client-side scripts. Create a class that extends `HtmlComponent` and place a `<slot />` where page content should appear.

```java MainLayout.java
package com.example.myapp;

import com.ciro.jreactive.HtmlComponent;

public class MainLayout extends HtmlComponent {

    @Override
    protected String template() {
        return """
            <!DOCTYPE html>
            <html lang="en">
            <head>
                <meta charset="UTF-8">
                <title>My App</title>
            </head>
            <body>
                <main id="app">
                    <slot />
                </main>

                <script src="/js/idiomorph.min.js"></script>
                <script src="/js/sockjs.min.js"></script>
                <script src="/js/jreactive-runtime.js"></script>
            </body>
            </html>
            """;
    }
}
```

The three scripts are served from `jreactive-runtime-js` — no CDN links or manual downloads required.

</Step>

<Step title="Create a counter component">

Components extend `HtmlComponent`. Annotate state fields with `@State` and event handlers with `@Call`. Bind events in the template using `@click`.

```java CounterPage.java
package com.example.myapp;

import com.ciro.jreactive.HtmlComponent;
import com.ciro.jreactive.State;
import com.ciro.jreactive.annotations.Call;
import com.ciro.jreactive.annotations.StatefulRam;
import com.ciro.jreactive.router.Layout;
import com.ciro.jreactive.router.Route;

@Route(path = "/counter")
@Layout(MainLayout.class)
@StatefulRam
public class CounterPage extends HtmlComponent {

    @State
    public int count = 0;

    @Call
    public void increment() {
        count++;
    }

    @Call
    public void reset() {
        count = 0;
    }

    @Override
    protected String template() {
        return """
            <div>
                <p>Count: <strong>{{count}}</strong></p>
                <button @click="increment()">+1</button>
                <button @click="reset()">Reset</button>
            </div>
            """;
    }
}
```

**What each annotation does:**

| Annotation | Effect |
|---|---|
| `@Route(path = "/counter")` | Maps HTTP `GET /counter` to this component |
| `@Layout(MainLayout.class)` | Wraps this page inside `MainLayout` |
| `@StatefulRam` | Stores component state in memory (Caffeine cache) per session |
| `@State` | Makes the field reactive — changes trigger a DOM diff push |
| `@Call` | Marks the method as callable from template event bindings |

Template syntax uses `{{fieldName}}` for interpolation and `@click="methodName(arg)"` for events. When a `@Call` method mutates a `@State` field, JReactive automatically pushes the diff to the browser over WebSocket.

</Step>

<Step title="Run the app and open the browser">

```bash
mvn spring-boot:run
```

Open [http://localhost:8080/counter](http://localhost:8080/counter). Click **+1** — the counter updates in place without a page reload.

The first request is served as fully rendered HTML (SSR). Subsequent interactions go over WebSocket: the browser sends a call event, the server executes the `@Call` method, computes the DOM diff, and pushes the changed HTML fragment back. The browser morphs the DOM using idiomorph.

</Step>

</Steps>

</Tip>
