title: ComponentAccessor<T>
description: The SPI interface that replaces reflection for reading fields, writing fields, and calling methods on a component.
keywords: [ComponentAccessor, AccessorRegistry, AOT, APT, reflection, JReactive]
---

`ComponentAccessor<T>` is the SPI interface that the JReactive AOT processor generates implementations of at build time. At runtime, the framework uses these generated classes instead of Java reflection to read fields, write fields, and invoke `@Call` methods — eliminating the performance and module-system overhead of reflection for hot paths.

```java
public interface ComponentAccessor<T extends HtmlComponent>
```

---

## Methods

### write(T component, String property, Object value)

```java
void write(T component, String property, Object value)
```

Sets a field on `component` by name. Replaces `Field.set()`.

<ParamField path="component" type="T" required>
  The component instance to write to.
</ParamField>

<ParamField path="property" type="string" required>
  The field name.
</ParamField>

<ParamField path="value" type="Object" required>
  The new value.
</ParamField>

### read(T component, String property)

```java
Object read(T component, String property)
```

Reads a field from `component` by name. Replaces `Field.get()`.

<ParamField path="property" type="string" required>

### call(T component, String method, Object... args)

```java
Object call(T component, String method, Object... args)
```

Invokes an `@Call`-annotated method on `component` by name. Replaces `Method.invoke()`.

### getAst()

```java
default java.util.List<com.ciro.jreactive.ast.JrxNode> getAst()
```

Returns the pre-compiled AST for this component's template. The AOT processor parses `template()` at build time and stores the result as a `List<JrxNode>` directly in the generated accessor class. At runtime, `AstComponentEngine` reads this list instead of re-parsing the template string, reducing startup latency and eliminating per-request parsing overhead.

Returns `null` if the component was not processed by the AOT APT (the engine falls back to parsing at runtime).

### getScopedCss()

```java
default String getScopedCss()
```

Returns the scoped CSS string for this component. The AOT processor reads the component's `.css` file at build time, scopes all selectors to the component's CSS scope class (e.g. `.jrx-sc-MyComponent`), minifies the result, and embeds it as a string literal in the generated accessor. At runtime `HtmlComponent._getBundledResources()` reads this string to inject the `<style>` block.

Returns `null` if no CSS file was found at build time.

## AccessorRegistry

`AccessorRegistry` is the global lookup table that maps component classes to their `ComponentAccessor` instances.

### AccessorRegistry.get(Class\<T\>)

```java
public static <T extends HtmlComponent> ComponentAccessor<T> get(Class<T> clazz)
```

Returns the `ComponentAccessor` for the given component class. On first call for a class, the registry attempts to load the generated class `ClassName__Accessor` from the classpath. Loading the class triggers its `static {}` block, which calls `register()`. Subsequent calls return the cached instance.

Returns `null` if no accessor class exists (the component will fall back to reflection).

### AccessorRegistry.register(Class\<T\>, ComponentAccessor\<T\>)

```java
public static void register(Class<?> clazz, ComponentAccessor<?> accessor)
```

Registers an accessor for a component class. Generated accessor classes call this from their `static {}` initializer block:

```java
// Generated by APT — do not edit
public class MyComponent__Accessor implements ComponentAccessor<MyComponent> {
    static {
        AccessorRegistry.register(MyComponent.class, new MyComponent__Accessor());
    }
    // ...
}
```

You can also call `register()` manually to provide a hand-written accessor for testing or advanced use cases.

### AccessorRegistry.has(Class\<?>\)

```java
public static boolean has(Class<?> clazz)
```

Returns `true` if an accessor is registered and loadable for the given class.

## How APT generates accessors

When you build your project with the JReactive annotation processor on the classpath, the APT generates a `ClassName__Accessor` class for every `HtmlComponent` subclass it finds. The generated class:

1. Implements `ComponentAccessor<ClassName>`.
2. Overrides `write()`, `read()`, and `call()` with direct field accesses and method calls — no reflection.
3. Overrides `getAst()` to return the pre-parsed `List<JrxNode>` for the component's template.
4. Overrides `getScopedCss()` to return the pre-scoped CSS string.
5. Registers itself in a `static {}` block via `AccessorRegistry.register()`.

<Note>
Accessor generation is optional but strongly recommended for production. Components without a generated accessor fall back to reflection automatically, which works correctly but is slower.
</Note>
